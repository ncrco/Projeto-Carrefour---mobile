# Guia de Contribuição

Obrigado por considerar contribuir para este projeto! Este documento fornece diretrizes para contribuições.

## Como Contribuir

### 1. Reportar Bugs

Se você encontrou um bug:
- Verifique se o bug já não foi reportado nas Issues
- Crie uma nova Issue descrevendo o problema
- Inclua passos para reproduzir o bug
- Adicione screenshots se aplicável

### 2. Sugerir Melhorias

Para sugerir melhorias:
- Crie uma Issue com a tag "enhancement"
- Descreva claramente a melhoria proposta
- Explique por que essa melhoria seria útil

### 3. Submeter Pull Requests

1. Faça um fork do projeto
2. Crie uma branch para sua feature (`git checkout -b feature/AmazingFeature`)
3. Faça commit das suas mudanças (`git commit -m 'Add some AmazingFeature'`)
4. Push para a branch (`git push origin feature/AmazingFeature`)
5. Abra um Pull Request

### Padrões de Código

- Siga o padrão Page Object já estabelecido
- Use nomes descritivos para variáveis e funções
- Adicione comentários quando necessário
- Mantenha os testes organizados e legíveis

### Estrutura de Commits

Use mensagens de commit descritivas:
- `feat: adiciona novo teste de login`
- `fix: corrige seletor do botão de submit`
- `docs: atualiza README com novas instruções`
- `refactor: reorganiza estrutura de Page Objects`

## Testes

Antes de submeter um PR:
- Certifique-se de que todos os testes passam
- Adicione testes para novas funcionalidades
- Mantenha a cobertura de testes

## Perguntas?

Se tiver dúvidas, abra uma Issue com a tag "question".
